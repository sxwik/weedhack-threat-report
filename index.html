▪ What it is

WeedHack Stealer is a sophisticated, multi-stage malware likely written in Java with possible secondary native/.NET payloads. It is purpose-built to extract session tokens, browser credentials, and cryptocurrency wallets from victims.

▪ Why it's dangerous

It leverages the trusted .jar format. Since players naturally execute .jar files for game mods, social engineering bypasses standard OS warnings. It executes silently within the JVM, leaving the game functional to avoid suspicion.

▪ Who is targeted

Victims range from casual players seeking FPS boosts, to server admins and mod developers. Attackers specifically aim to hijack Microsoft accounts with Minecraft licenses and Discord accounts with server admin privileges.

Origin & Distribution

Understanding how WeedHack propagates is crucial for defense. This section explores the origins of the malware within underground forums and visualizes the primary channels attackers use to socially engineer victims into downloading the malicious payload.

Threat Origins

Investigation indicates WeedHack Stealer is believed to have emerged in underground cheat communities in late 2024. It was initially marketed as a MaaS (Malware-as-a-Service) builder known under aliases like "BlockGrabber" and "CraftSteal". The developers realized that bypassing Windows Defender was significantly easier when the payload was bundled within a compiled Java class rather than a traditional executable.

Primary Vectors

  • 1.
    Discord Servers: Threat actors compromise prominent community Discord servers, posting malicious links in "announcement" or "mod-release" channels.
  • 2.
    YouTube Tutorials: Fake videos offering "FPS Boosters", "Unban Tools", or "Free Premium Cheats" with malicious Mega/MediaFire links in the description.
  • 3.
    GitHub Repositories: Forking legitimate open-source mods, injecting the loader, and artificially inflating stars to appear legitimate.

Technical Malware Analysis

This section details the inner workings of WeedHack Stealer. To manage the technical depth, the execution chain is broken down into sequential phases. Click on each phase to explore the specific files, obfuscation techniques, and APIs abused by the malware to achieve persistence and evasion.

File Structure: The initial vector is a `.jar` file (e.g., OptiFine_1.19.4_HD_U_I8.jar). Inside, alongside legitimate mod files, lies a heavily obfuscated class (often named randomly like a/b/c.class or utilizing zero-width characters).

Obfuscation: WeedHack uses custom string encryption and flow obfuscation (similar to ProGuard but heavily modified) to hide its payload. Security analysts must deobfuscate using tools like CFR or Krakatau, though strings are decrypted at runtime using a hardcoded XOR key.

Execution: When the user launches Minecraft, the JVM loads the mod. The malicious class utilizes the `java.lang.Runtime.getRuntime().exec()` method to quietly drop the secondary payload.

Payload Extraction: The Java loader extracts a base64 encoded string from its resources, decodes it into a `.zip` archive, and extracts the core executable (often named javaw_updater.exe or sys_diag.exe) into the %TEMP% or %APPDATA%\.minecraft\versions directory.

Anti-VM & Anti-Debug: Before running core functions, the payload checks for VirtualBox, VMware MAC addresses, and queries `WMI` for sandbox environments. If a debugger is attached (checking `IsDebuggerPresent` API), it terminates immediately.

WeedHack aims to survive reboots to continue siphoning updated tokens. It achieves this via two primary methods:

  • Registry Modification: Writes itself to HKCU\Software\Microsoft\Windows\CurrentVersion\Run under a deceptive name like "Java Update Scheduler".
  • Scheduled Tasks: Uses `schtasks.exe` to create a hidden task triggering execution on user logon.

It avoids creating Windows Services, likely because doing so requires higher privileges that standard Minecraft players might not grant to a game mod.

Credential Theft & RAT Capabilities

What exactly does WeedHack steal? This section quantifies the data targeted by the infostealer module and provides a critical assessment of the malware's backdoor functionalities. We separate confirmed behaviors from community speculation to provide accurate intelligence.

Targeted Data Priority (By Frequency)

  • Minecraft/Microsoft Tokens: Targets launcher_accounts.json and extracts Xbox Live auth tokens, allowing session hijacking without passwords.
  • Discord Tokens: Scans LocalAppData for Discord, PTB, and Canary LevelDB files. Uses Regex to extract user tokens.
  • Browser Data: Injects into Chrome/Edge/Firefox processes to decrypt the `Login Data` SQLite database, stealing saved passwords, autofill, and active session cookies.
  • Crypto Wallets: Hunts for extensions like MetaMask, Phantom, and desktop wallets (Exodus).
  • System Info: Captures a screenshot upon execution, IP address, hardware specs, and running process list.
CONFIRMED Secondary Payloads: WeedHack contains logic to download and execute arbitrary executables from its C2 server upon command. This is primarily used to update itself or drop cryptominers.
SUSPECTED / RARE Live Shell Access: While some forums claim WeedHack gives attackers live remote desktop access, reversing of the primary variants shows no VNC or reverse-shell capabilities. It is primarily a "grab and go" stealer.
UNVERIFIED Keylogging: No active `SetWindowsHookEx` API calls for systemic keylogging were found in the core analysis, though modular versions may exist.

Infrastructure & IOCs

WeedHack relies heavily on abused legitimate services (Webhooks) rather than dedicated C2 servers for exfiltration, making network-level blocking difficult. Below are the confirmed Indicators of Compromise (IOCs) for threat hunting and firewall configuration.

Exfiltration Mechanism

Data is compressed into an encrypted ZIP file and sent via HTTP POST to Discord Webhooks or Telegram Bot APIs. Dedicated C2s (like weed-panel[.]ru) are only used for license validation of the malware builder.

Network Signatures

Look for unusually large outbound HTTP POST requests to discord.com/api/webhooks/ originating from javaw.exe or unknown executables in the Temp folder.

Detection & Removal Guide

If you suspect an infection, immediate action is required. Because session tokens bypass passwords, simply changing your password is not enough. Follow this structured remediation workflow to sever the attacker's access and clean the system.

1 Containment (Do This First)

Disconnect from the internet immediately. This prevents the malware from sending newly generated tokens to the attacker while you clean the system.

2 Detection & Eradication

Antivirus Scans: Run a full system scan using Windows Defender or Malwarebytes. Look for detections flagged as Trojan:Win32/Wacatac, MSIL/AgentTesla, or generic Java droppers.

Manual File Check: Navigate to %APPDATA%\.minecraft\mods and %APPDATA%\.minecraft\versions. Delete any .jar files you did not explicitly download from a trusted source. Delete the entire %TEMP% folder contents.

Check Persistence: Open Task Manager -> Startup. Disable anything suspicious (e.g., "Java Updater", blank names). Press Win+R, type taskschd.msc, and look for anomalous scheduled tasks running hidden executables.

3 Revocation & Recovery (Crucial Step)

Warning: Because WeedHack steals Session Cookies, attackers can stay logged into your accounts even if you change the password. You must invalidate the session.
  • Discord: Go to User Settings -> Devices -> Scroll to bottom -> "Log Out All Known Devices". Change your password.
  • Minecraft/Microsoft: Log into Microsoft Account Security. Go to "Advanced security options" -> "Sign me out everywhere". Change your password.
  • Browsers: In Chrome/Edge/Firefox, clear ALL browsing data (specifically Cookies and cached files) to invalidate stolen session tokens.
  • Wallets: If you had a crypto wallet extension installed, consider that wallet compromised. Transfer funds to a newly created wallet on a clean device immediately.
